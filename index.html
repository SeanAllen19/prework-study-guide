HTML

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource. The href attribute is using a relative path. This means the location is derived from the current location (in this case, index.html). To locate the style sheet from the current location, we must look in the assets folder, hence assets/style.css.

Git

  • git status: checks what branch we are currently on
  • git add: add a file as it looks now to your next commit (stage)
  • git commit -m "(Detailed message)": commit your staged content as a new commit snapshot
  • git checkout: switch to another branch and check it out into your working directory
  • git checkout -b branch-name: creates a new branch and switches to it
  • git push [alias] [branch]: Transmit local branch commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch

Git Work Flow


  1. In the command line, make sure to be in the right directory. Use the cd command in the CLI to get to the proper directory (cd bootcamp/prework-study-guide)
  2. Once in the proper directory use the "code ." Command to start working in the project. Make sure to look at the files and make sure that all the files are there.
  3. Check the status of what branch you are working on. To switch to a different branch so you don’t change the MAIN branch. Use the command: git checkout -b (feature/starter-code) (The feature/starter-code is the name of the new branch. We have added the prefix, feature, to indicate that it is a feature. It is good practice to name the branches for the feature that will be developed by them to help indicate the purpose of each branch.)
  4. Use git status to make sure that you are working on the correct working branch. If all is according to plan, use git add -A, to add all the modifications from the working branch to the staging area. The -A flag indicates that we want to add all changes. 
  5. Before we push this up to our repo, we should double-check if our local branch is in sync with the base branch in GitHub. So let’s pull down any changes from the base branch with the command “ git pull origin main ”. We identify a Git command with the git keyword. Then use the pull command to receive a branch's modifications into the local environment. The origin keyword indicates the source of the pull will be in the GitHub repo. The main keyword indicates the branch. In our case, we are receiving a branch called main from the GitHub repository. It is important to always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work. Otherwise, you might be working on outdated code!
  6. Now our local branch has synced with the remote base branch, let's push our changes to the remote branch by using the following command: git push origin feature/starter-code

TL/DR

  1. Once in the proper working directory on your computer:
  2. git add -A
  3. git commit -m “your message here”
  4. git pull origin main
  5. git push origin feature/add-js
